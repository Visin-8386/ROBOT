ESP32-CAM chụp ảnh JPEG gửi qua TCP. Server chạy YOLO phát hiện người, tính pan/tilt, gửi qua MQTT. ESP32 Motor nhận lệnh MQTT điều khiển motor, gửi cảnh báo sensor. Web Dashboard và Flutter App giao tiếp qua REST API.
2. Luồng dữ liệu chính
sequenceDiagram
    participant CAM as ESP32-CAM
    participant SRV as Server TCP
    participant DET as YOLO Detector
    participant MQTT as MQTT Broker
    participant ESP as ESP32 Motor
    participant DB as PostgreSQL
    participant TG as Telegram

    Note over CAM,TG: LUONG PHAT HIEN NGUOI/THU CUNG

    loop Moi frame 25fps
        CAM->>SRV: [4B length][JPEG data] TCP :8765
        SRV->>SRV: Decode JPEG to OpenCV frame
        SRV->>DET: frame_queue.put(frame)
    end

    loop Main detection loop
        DET->>DET: 1. Tien xu ly Denoise + CLAHE
        DET->>DET: 2. YOLOv8 inference
        DET->>DET: 3. Loc muc tieu (nguoi/thu cung) conf>=0.5
        DET->>DET: 4. SORT & Kalman Filter (Lock ID + Confirmed)

        alt Phat hien muc tieu confirmed
            DET->>MQTT: Publish robot/position
            MQTT->>ESP: Forward position data
            DET->>DB: Save DetectionEvent + screenshot
            DET->>TG: Send photo cooldown 30s
        else Khong phat hien
            DET->>MQTT: Publish detected=false
        end
    end

    Note over CAM,TG: LUONG CANH BAO SENSOR

    loop Moi 50ms
        ESP->>ESP: Doc VL53L0X + PIR sensor
        alt PIR hoac Distance alert
            ESP->>MQTT: Publish robot/alert
            MQTT->>SRV: Forward alert
            SRV->>DB: Save SensorAlert
        end
    end

    Note over CAM,TG: LUONG DIEU KHIEN

    Note right of DB: User gui lenh tu Web/App
    DB->>SRV: POST /api/control?action=xxx
    SRV->>MQTT: Publish robot/command
    MQTT->>ESP: Forward command
    ESP->>ESP: Thuc thi motor command
3 luồng chính: (1) Phát hiện người: ESP32-CAM → TCP → YOLO → MQTT → ESP32 (2) Cảnh báo sensor: ESP32 → MQTT → Server → DB (3) Điều khiển: Web/App → API → MQTT → ESP32
3. State Machine — ESP32 Motor Controller

    PATROL --> CHASE: Camera AI phat hien muc tieu
    PATROL --> MONITOR: Web cmd monitor
    PATROL --> MANUAL: Web cmd forward/back/left/right

    CHASE --> PATROL: Mat tin hieu muc tieu hon 3s
    CHASE --> MONITOR: Web cmd monitor

    MANUAL --> MONITOR: Timeout 2s khong co lenh
Trạng thái Hành vi Chuyển trạng thái
MONITOR Motor DỪNG. Giám sát PIR + VL53L0X. Gửi alert mỗi 5-10s. → PATROL/CHASE/MANUAL qua web command
PATROL Tự tuần tra speed=150. Tránh vật cản <300mm (rẽ 800ms). → CHASE (phát hiện người), → MONITOR (web cmd)
CHASE Đuổi theo người speed=200. Dừng khi <500mm. Rẽ theo pan angle. → PATROL (mất người >3s), → MONITOR (web cmd)
MANUAL Điều khiển trực tiếp WASD từ web dashboard. → MONITOR (timeout 2s không có lệnh)

4. Detection Pipeline — Server
flowchart LR
    A["JPEG tu\nESP32-CAM"] --> B["cv2.imdecode\nBGR frame"]
    B --> C{"Tien xu ly?"}
    C -->|Co| D["GaussianBlur"]
    D --> E["CLAHE"]
    E --> F["Unsharp Mask"]
    F --> G["YOLOv8n\nInference"]
    C -->|Khong| G
    G --> I["Loc muc tieu\nnguoi/thu cung\nconf>=0.5"]
    I --> J{"Co muc tieu?"}
    J -->|"Co"| K["Chon muc tieu lon nhat\n(hoac theo ID)"]
    K --> L["SORT Tracker\n(Kalman + Hungarian)"]
    L --> M{">= 3 frame?"}
    M -->|"Confirmed"| N["MQTT + DB\n+ Telegram\n+ Servo"]
    M -->|"Chua du"| O["MQTT publish\nchua confirmed"]
    J -->|"Khong"| P["Kalman Filter\n(Predict)"]
    P --> Q{"Qua max_age\n(Mat dau)?"}
    Q -->|"Co"| R["Xoa ID, \nconfirmed=false"]
    Q -->|"Chua"| S["Giu trang\nthai cu"]
    R --> T["MQTT\ndetected=false"]
    S --> T
    style A fill:#f59e0b,color:#000
    style G fill:#8b5cf6,color:#fff
    style N fill:#10b981,color:#fff
    style L fill:#3b82f6,color:#fff
Pipeline: JPEG Decode → Tiền xử lý (GaussianBlur + CLAHE + Unsharp Mask) → YOLOv8 inference → Lọc person (class=0, conf≥0.5) → SORT + Kalman Tracker (≥3 frame = confirmed) → Publish + Save + Notify.
